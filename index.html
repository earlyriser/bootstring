How it works

Babelwhale is completely flexible, it's simple a collection of json files mapping the basic words and phrases an internationalized app needs. Just download it from Github and adapt it as you need.

A language file looks like this:
{
"hello":"Hola",
"profile":"Perfil"
}

Benefits

If you internationalize your app from the start you're getting:

  • Better code: Even if you are not launching your app in multiple countries, internationalizing from the start will help you decoupling your code.
  • Robust UI: Dealing with strings of different length, will make your UI more flexible and able to deal with future changes.
  • More users: If you're brave enough to launch internationally, you'll get more visits.
